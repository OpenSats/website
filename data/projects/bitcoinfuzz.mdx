`bitcoinfuzz` is a differential fuzzing harness for Bitcoin implementations and libraries. It builds multiple modules side by side, feeds them the same randomized inputs, and compares how they parse, validate, and serialize data. That catches crashes, consensus edge cases, and behavior mismatches before they ship downstream.

The public repository already covers a broad set of Bitcoin, Lightning, and library modules, along with a separate public [corpora](https://github.com/bitcoinfuzz/corpora) repository for shared fuzz inputs. The reports show why that matters. Earlier work exposed a critical NBitcoin consensus bug through script evaluation. The latest public report adds differential `libbitcoinkernel` modules, the first Utreexo target with `rustreexo`, `libwally-core` integration that found three bugs including a BIP32 non-conformance, `bitcoin-s`, and a new `verify_script` target across Bitcoin Core, `btcd`, and NBitcoin.

## Why fund it?

Bitcoin has multiple full nodes, libraries, bindings, and wallet stacks. That diversity needs constant cross-checking. `bitcoinfuzz` turns implementation differences into concrete test cases that maintainers can inspect and fix.

OpenSats highlighted `bitcoinfuzz` in its [grant announcement for Bruno Garcia][announcement]. Support for this work helps fund maintenance, new targets, new module integrations, shared corpora, and the build infrastructure needed to keep cross-implementation testing useful as the ecosystem changes.

## What's next?

The latest public report gives a concrete roadmap. Next work includes `libbitcoin-system` coverage for block, script, and transaction parsing, more Utreexo targets, PSBT v2 support in the `psbt_parse` harness, a full `ellswift` roundtrip target covering `btcd` and `secp256k1`, and a MuSig2 target.

Infrastructure work is moving too. The recent report notes that Bitcoin Core moved to a submodule, multi-module Go build conflicts were fixed, and Docker CI setup kept improving. That keeps the harness easier to update while the target set grows.
