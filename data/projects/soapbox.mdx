Soapbox started as an alternative Mastodon frontend but has grown into a family
of projects spanning nostr and the fediverse. The main pieces are [Soapbox]
itself (a customizable, self-hostable social media UI), [Ditto] (a self-hosted
nostr server with a built-in client and relay, compatible with Mastodon apps),
and the [Mostr] bridge, which connects nostr to the ActivityPub fediverse
bidirectionally.

Ditto gives communities a way to run their own social server on nostr while
staying interoperable with Mastodon through Mostr and with Bluesky through
[Eclipse]. Users get zaps, DMs, NIP-05 username purchases, and a digital shop
for profile customization, all from a single self-hosted instance.

Alex also maintains [Nostrify], a nostr framework for web browsers and [Deno]
that other developers can build on. He's also building [Shakespeare].

## Why fund it?

Bridging nostr and the fediverse expands the reach of both networks. The Mostr
bridge alone exposes millions of fediverse users to nostr, and several have
already moved over entirely. Ditto makes it possible to run a nostr-first server
that still talks to the rest of the social web, lowering the barrier for
communities that want to try nostr without cutting ties to existing networks.

OpenSats has been supporting Soapbox since the [first wave of nostr
grants][wave-1] in July 2023. Alex later received a [long-term support
grant][lts] in August 2024.

## What's next?

Ditto reached a stable release and powers community instances around the world,
including [cobrafuma.com] in Brazil, [seewaan.com] for Arabic speakers, and
[cvrkut.com] in Croatia. Recent versions added Cashu wallet support, machine
translations, and video transcoding. Alex also moved the project's
infrastructure off the cloud and onto a self-hosted server rack.

Alex's recent focus is [Shakespeare], an open-source tool for building Nostr
apps by talking to an AI. It runs in the browser, stores files locally, and
works with any AI provider the user chooses. People with no programming
experience have already used it to build working clients. Alex has been running
workshops at Bitcoin conferences to get it into more hands. The longer-term plan
is to bring this back to Soapbox: letting non-technical community leaders set up
and customize their own servers without touching a terminal.
