Soapbox started as an alternative Mastodon frontend but has grown into a family
of projects spanning nostr and the fediverse. The main pieces are
[Soapbox](https://docs.soapbox.pub/soapbox/) itself (a customizable,
self-hostable social media UI), [Ditto](https://soapbox.pub/ditto/) (a
self-hosted nostr server with a built-in client and relay, compatible with
Mastodon apps), and the [Mostr](https://soapbox.pub/blog/mostr-fediverse-nostr-bridge/)
bridge, which connects nostr to the ActivityPub fediverse bidirectionally.

Ditto gives communities a way to run their own social server on nostr while
staying interoperable with Mastodon through Mostr and with Bluesky through
[Eclipse](https://eclipse.pub/). Users get zaps, DMs, NIP-05 username
purchases, and a digital shop for profile customization, all from a single
self-hosted instance.

## Why fund it?

Bridging nostr and the fediverse expands the reach of both networks. The Mostr
bridge alone exposes millions of fediverse users to nostr, and several have
already moved over entirely. Ditto makes it possible to run a nostr-first server
that still talks to the rest of the social web, lowering the barrier for
communities that want to try nostr without cutting ties to existing networks.

OpenSats has been supporting Soapbox since the [first wave of nostr
grants](/blog/nostr-grants-july-2023#soapbox) in July 2023. Alex later received
a [long-term support grant](/blog/alex-gleason-receives-lts-grant) in August 2024.

## What's next?

Ditto reached a stable release and powers community instances around the world,
including [cobrafuma.com](https://cobrafuma.com) in Brazil,
[seewaan.com](https://seewaan.com) for Arabic speakers, and
[cvrkut.com](https://cvrkut.com) in Croatia. Recent versions added Cashu wallet
support, machine translations, and video transcoding. Alex also moved the
project's infrastructure off the cloud and onto a self-hosted server rack.

Alex's recent focus is [Shakespeare](https://shakespeare.diy/), an open-source
tool for building Nostr apps by talking to an AI. It runs in the browser,
stores files locally, and works with any AI provider the user chooses. People
with no programming experience have already used it to build working clients.
Alex has been running workshops at Bitcoin conferences to get it into more
hands. The longer-term plan is to bring this back to Soapbox: letting
non-technical community leaders set up and customize their own servers without
touching a terminal.
