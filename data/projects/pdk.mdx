The [Payjoin Dev Kit](https://payjoindevkit.org/) (PDK) is the reference
Rust implementation of [Payjoin](/topics/payjoin), the cooperative on-chain
transaction protocol that breaks the
[common-input-ownership heuristic](https://en.bitcoin.it/wiki/Privacy#Common-input-ownership_heuristic)
chain analysis relies on. PDK ships the
[rust-payjoin](https://github.com/payjoin/rust-payjoin) crate along with
FFI bindings for Python, Dart, Swift, Kotlin, JavaScript, and C#, so any
wallet or service can add Payjoin by dropping in one library instead of
each team reimplementing the protocol.

PDK is stewarded by the [Payjoin Foundation](https://payjoin.org/), a
501(c)(3) non-profit incorporated in Delaware in 2025 and led by
[Dan Gould](https://github.com/DanGould). The dev kit powers
[BIP 77](https://github.com/bitcoin/bips/blob/master/bip-0077.md),
Async Payjoin, which uses
[Oblivious HTTP](https://www.rfc-editor.org/rfc/rfc9458.html) so sender and
receiver can exchange messages without both being online at once. That
removed the biggest practical barrier to adoption: Payjoin v1 needed the
receiver to run a public server, often behind Tor and TLS, which very few
wallets ever did.

## Why fund it?

Bitcoin privacy on-chain needs widespread adoption of tools that actively
break surveillance heuristics. A clean, production-ready library gives wallet
and service developers a drop-in way to add this behavior by default, without
each project building its own implementation.

OpenSats started funding this work as the Payjoin Dev Kit in the
[first wave of Bitcoin grants](/blog/bitcoin-grants-july-2023#payjoin-dev-kit)
in July 2023, and renewed the grant in the
[eleventh wave](/blog/eleventh-wave-of-bitcoin-grants) and again in the
[seventeenth wave](/blog/seventeenth-wave-of-bitcoin-grants), now directly awarded to
the Payjoin Foundation. For a longer write-up, see the impact reports on
[Async Payjoin](/blog/developing-advancements-in-onchain-privacy#async-payjoin)
and the
[Payjoin Dev Kit](/blog/advancements-in-developer-libraries#payjoin-dev-kit).

## What's next?

[Bull Bitcoin](https://bullbitcoin.com/) shipped Payjoin v2 in December 2024
and [Cake Wallet](https://cakewallet.com/) followed in May 2025.
[BDK-CLI](https://github.com/bitcoindevkit/bdk-cli) shipped v3.0.0 with
Payjoin support in early 2026. Pull requests are open for
[Liana](https://github.com/wizardsardine/liana/),
[LDK-Node](https://github.com/lightningdevkit/ldk-node),
[Boltz](https://boltz.exchange/), and [LEXE](https://lexe.app/), and a
[BTCPayServer](https://btcpayserver.org/) plugin is in development.

The `payjoin` crate has moved from version 0.11 to
[1.0-rc.2](https://github.com/payjoin/rust-payjoin/releases/tag/payjoin-1.0.0-rc.2),
adding persistence baked into the core state machine, a session-event log so
apps can show transaction history, and stable FFI for every binding target. [`payjoin-mailroom`](https://github.com/payjoin/rust-payjoin/tree/master/payjoin-mailroom)
bundles the directory, OHTTP relay, and metrics into a single binary that
[BOB Space](https://bobspaces.net/), Cake Wallet, Vinteum, Achow101, and
others now run as public infrastructure.

Alongside the dev kit, the Foundation runs a research track led by 
[Armin Sabouri](https://github.com/arminsabouri) on wallet fingerprinting
([tx-indexer](https://github.com/payjoin/tx-indexer),
[btsim](https://github.com/payjoin/btsim), and an automated
[RAG pipeline for wallet code](https://github.com/arminsabouri/WalletProbe)),
publishes open [weekly dev-call notes](https://github.com/orgs/payjoin/discussions/categories/dev-call-notes)
and a [study club](https://github.com/orgs/payjoin/discussions/categories/study-club),
and is drafting a multiparty Payjoin v3 protocol with multiple receivers.
