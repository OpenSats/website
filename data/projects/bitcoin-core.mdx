Bitcoin Core is the full node software that validates blocks and transactions, enforces consensus rules, relays data across the peer-to-peer network, and lets users verify Bitcoin for themselves.

OpenSats began funding Bitcoin Core contributors in 2023 with its [Long-Term Support program](/blog/announcing-lts-grant-program-to-support-bitcoin-core-contributors), which first backed [Marco Falke](/blog/announcing-lts-grant-program-to-support-bitcoin-core-contributors). That program later funded [Josie Baker](/blog/josi-baker-receives-opensats-lts-grant), [Sjors Provoost](/blog/sjors-provoost-receives-opensats-lts-grant), [Vasil Dimov](/blog/vasil-dimov-receives-lts-grant), [Gleb Naumenko](/blog/gleb-naumenko-receives-lts-grant), [Furszy](/blog/furszy-receives-lts-grant-for-bitcoin-core), [Will Clark](/blog/will-clark-receives-lts-grant-for-bitcoin-core), [0xB10C](/blog/0xB10C-receives-lts-grant), [Bruno Garcia](/blog/bruno-garcia-receives-lts-grant), [Jon Atack](/blog/jon-atack-receives-lts-grant), and [Andrew Toth](/blog/andrew-toth-receives-lts-grant).

## Why fund it?

Bitcoin Core does not maintain itself. It needs people who stay with review, testing, and maintenance over months and years. Bugs hide in edge cases, old code, slow machines, and unexpected interactions between features.

Most of the day-to-day work is invisible: release testing, CI and fuzzing upkeep, debugging intermittent failures, benchmarking on different hardware, reviewing other people's pull requests. Funding lets contributors do that work full-time. It turns one-off fixes into sustained maintenance.

## What's next?

Review is the bottleneck. Large pull requests sit for months waiting for someone who can read them closely, test them on real hardware, and follow through across rebases and revisions.

Andrew Toth is working on a block validation speedup that would improve both Initial Block Download and processing at the chain tip. L0rinc is benchmarking Core on everything from cloud servers to Raspberry Pis and looking at ways to cut memory usage. Marco Falke maintains the test framework and CI infrastructure the rest of the project depends on.

Private transaction broadcast merged this year. Toth has follow-up work to improve reliability across restarts and give node operators more control over the feature.

Sjors Provoost maintains the Bitcoin Core side of Stratum V2, fixing bugs surfaced by SRI testing and building out the IPC methods mining software needs. He is also working on MuSig2 support through HWI and reviewing the BIP54 consensus cleanup.

The libbitcoinkernel project is pulling Core's consensus engine into a standalone library. janb84 joined a working group tackling the global `cs_main` lock, replacing it with finer-grained locking so third-party applications can embed the kernel without running a full node.

On the network side, Julien Urraca brought ASMap generation under ten minutes and coordinated collaborative runs to produce canonical ASmaps for Core. Naiyoma is researching P2P fingerprinting defenses.

Build tooling, CI, fuzzing, and release engineering touch every release but rarely get attention. The codebase has to stay safe to change, and that takes people who show up for it.
