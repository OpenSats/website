Bitcoin Core is the full node software that validates blocks and transactions, enforces consensus rules, relays data across the peer-to-peer network, and lets users verify Bitcoin for themselves.

That work depends on contributors who review pull requests, write and run tests, fix bugs, improve performance, maintain wallet and RPC code, and keep the codebase legible enough for new contributors to join.

## Why fund it?

Bitcoin Core does not maintain itself. It needs steady review, testing, maintenance, and contributor development. That work is slow, technical, and easy to underfund.

OpenSats has funded Bitcoin Core contributors since launching its [LTS program](/blog/announcing-lts-grant-program-to-support-bitcoin-core-contributors) in 2023. Since then, that support has expanded to maintainers, reviewers, testers, and newer contributors working on Initial Block Download, release testing, test coverage, build system maintainability, the [Bitcoin Core Testing Suite](/blog/bitcoin-grants-feb-2024#bitcoin-core-testing-suite), and peer-to-peer privacy research.

## What's next?

The backlog is still deep. Review remains a bottleneck. Release testing, testing infrastructure, `libbitcoinkernel`, package relay, wallet work, sync performance, and peer-to-peer privacy all need sustained attention.

For a fuller picture of OpenSats support around Bitcoin Core, see [Caring for Bitcoin Core](/blog/caring-for-bitcoin-core), [Additional Grants for Bitcoin Core Contributors](/blog/more-for-core), and [5 Grants to Strengthen Bitcoin Development](/blog/five-grants-to-strengthen-bitcoin-development).
