Bitcoin Core is the reference implementation of the Bitcoin protocol. It is the full node software that validates blocks and transactions, enforces consensus rules, relays data across the peer-to-peer network, and gives users a way to verify the system for themselves.

Core contributors review and test pull requests, fix bugs, improve performance, maintain wallet and RPC interfaces, and keep the codebase understandable enough for new contributors to join. Bitcoin depends on that work.

## Why fund it?

Bitcoin Core needs sustained review, testing, maintenance, and onboarding. That requires experienced contributors and enough support for newer developers to build context over time.

OpenSats supports Bitcoin Core through multiple channels: long-term support grants for individual contributors, targeted funding for review and testing work, and grants for adjacent efforts such as the [Bitcoin Core App](/blog/caring-for-bitcoin-core#bitcoin-core-app) and the [Bitcoin Core Testing Suite](/blog/bitcoin-grants-feb-2024#bitcoin-core-testing-suite). Our support began with the [LTS program for Bitcoin Core contributors](/blog/announcing-lts-grant-program-to-support-bitcoin-core-contributors) and has continued through later grant rounds.

## What's next?

The work is ongoing: review, testing, tooling, clearer interfaces, and continued effort to reduce bottlenecks in the contributor pipeline. Areas like `libbitcoinkernel`, package relay, wallet improvements, sync performance, and testing infrastructure all need steady attention.

If you want a broader picture of the work OpenSats is helping fund around the project, start with [Caring for Bitcoin Core](/blog/caring-for-bitcoin-core) and [Additional Grants for Bitcoin Core Contributors](/blog/more-for-core).
