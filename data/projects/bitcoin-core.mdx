Bitcoin Core is the full node software that validates blocks and transactions, enforces consensus rules, relays data across the peer-to-peer network, and lets users verify Bitcoin for themselves.

Keeping it healthy means more than writing new features. The reports show contributors doing release testing, code review, CI and fuzzing work, debugging intermittent failures, benchmarking performance across different machines, improving wallet and RPC behavior, and maintaining interfaces that other projects build on.

## Why fund it?

Bitcoin Core does not maintain itself. It needs review, testing, maintenance, and time for contributors to build context. That work is slow, technical, and usually underfunded.

OpenSats has funded Bitcoin Core contributors since launching its [LTS program](/blog/announcing-lts-grant-program-to-support-bitcoin-core-contributors) in 2023. The reports now cover a wide range of work: release QA, nightly fuzzing and coverage, the [Bitcoin Core Testing Suite](/blog/bitcoin-grants-feb-2024#bitcoin-core-testing-suite), IBD and chainstate performance, private transaction broadcast, fingerprinting and ASMap research, build system work, and libbitcoinkernel-adjacent tooling.

This is the kind of work that can sit in review queues for months or years unless someone can stay on it. The reports show contributors benchmarking across platforms, splitting large changes into smaller pieces, writing follow-up fixes and tests after features land, and maintaining the tooling that helps catch regressions earlier.

## What's next?

Review is still the choke point. Marco Falke keeps running and improving CI, nightly fuzzing, coverage, and release QA. `janb84`’s reports point to the same bottleneck through release testing, contributor docs, and review. Kevin’s current grant is focused on Core maintenance and the testing suite.

Performance work is still moving. Andrew Toth is pushing validation and block-processing improvements around `#31132`, alongside follow-up work on private transaction broadcast. Lőrinc Pap is working on IBD, chainstate, coins cache maintenance, benchmarking, and secp256k1 optimization.

Network privacy and peer management still need work. Private transaction broadcast has landed, but the fixes, RPC follow-ups, and tests are still being worked through. Naiyoma’s current grant focuses on fingerprinting attacks. Julien Urraca and Hodlinator have both been pushing ASMap forward.

Maintainability keeps surfacing in the reports. Daniel Pfeifer is working on BtcK, CMake, CI with CTest and CDash, and libbitcoinkernel-adjacent work. Marco’s recent reports also cover undefined behavior fixes, valgrind, fuzz determinism, and hard-to-reproduce infrastructure bugs.

Mining, wallet, and interfaces still need work as well. Sjors Provoost is fixing issues exposed by Stratum V2 testing, reviewing IPC changes, and landing related Core changes. Daniela Brozzoni’s current grant adds more implementation and review capacity.

For a fuller picture of OpenSats support around Bitcoin Core, see [Caring for Bitcoin Core](/blog/caring-for-bitcoin-core), [Additional Grants for Bitcoin Core Contributors](/blog/more-for-core), and [5 Grants to Strengthen Bitcoin Development](/blog/five-grants-to-strengthen-bitcoin-development).
