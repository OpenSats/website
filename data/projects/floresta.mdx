[Floresta](https://getfloresta.org/) is a Bitcoin full node written in Rust.
It keeps storage and memory requirements low by combining
[Utreexo](https://eprint.iacr.org/2019/611) (a cryptographic accumulator that
replaces the UTXO set with a small set of Merkle roots) with
[PoW Fraud Proofs](https://blog.dlsouza.lol/2023/09/28/pow-fraud-proof.html)
and pruning. The goal is to let people verify Bitcoin on hardware they already
own, including phones and small single-board computers, without giving up
trust-minimization.

The project ships two main pieces:
[`libfloresta`](https://github.com/vinteumorg/Floresta/tree/master/crates),
a set of reusable crates that applications can embed, and
[`florestad`](https://github.com/vinteumorg/Floresta/tree/master/bin/florestad),
a daemon that runs as a standalone node with a view-only wallet and a built-in
Electrum server. Developers can use `libfloresta` the way they would use
[`rust-bitcoin`](https://github.com/rust-bitcoin/rust-bitcoin) or
[`bdk`](/projects/bdk), building wallets, explorers, and indexers on top of a
validating client.

## Why fund it?

Running a full node is the strongest form of trust-minimization on Bitcoin.
Resource requirements push most users onto custodial services or light clients
that trust third parties. Floresta does real validation with a footprint small
enough to run on commodity hardware, which makes it a good fit for phones,
single-board computers, and embedded wallet stacks.

OpenSats first funded Floresta in the
[seventh wave of Bitcoin grants](/blog/bitcoin-grants-september-2024-7th-wave#floresta)
in September 2024. The [fourteenth wave](/blog/fourteenth-wave-of-bitcoin-grants#floresta)
added grants for contributors
[João Leal](https://github.com/jaoleal),
[Moises Pompilio](https://github.com/moisesPompilio), and
[Jose Castillo López](https://github.com/jose-sastre). The project lives at
[vinteumorg/Floresta](https://github.com/vinteumorg/Floresta), alongside other
Bitcoin research and engineering work at [Vinteum](https://vinteum.org/).

## What's next?

Recent work focused on expanding `florestad`'s RPC surface to match Bitcoin
Core more closely, so existing tooling can talk to a Floresta node without
rewrites. Other threads include packaging for
[nix-bitcoin](https://nixbitcoin.org/) and upstream
[nixpkgs](https://github.com/NixOS/nixpkgs), contributor onboarding, code
review, and improvements to the testing and fuzzing setup. `libfloresta` keeps
gaining consumers as more wallets and indexers embed Floresta directly.
